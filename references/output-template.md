# Output Template

Use this structure for the final answer. Compress it when the user wants a shorter read, but keep the same logic.

## Market and Data

- Instrument
- Timeframe
- Date range analyzed
- Data source
- EMAs used
- Key levels

## Brooks Read

- Market cycle
- Always-in direction
- Why the current bars are strong or weak
- What the EMA relationship adds to the read

## Bull Case

- Trigger
- Why it can work
- Invalidation
- First target
- Stretch target

## Bear Case

- Trigger
- Why it can work
- Invalidation
- First target
- Stretch target

## Trade Plan

- Preferred action now: long, short, wait, or reduce risk
- Entry style: market, stop entry, limit on pullback, or no trade
- Stop logic
- Risk note

## What Changes The Read

- One condition that upgrades the bull case
- One condition that upgrades the bear case
- One condition that keeps the chart in no-trade mode

## Language rules

- Tie opinions to bars, levels, or EMA relationships.
- Do not sound certain in a range or during a weak breakout.
- If the user asks for an investment suggestion, keep it scenario-based and not personalized.
